Write a paper on a fight: Superman (New52) vs King Hyperion (Earth-4023)

Create one scan-first dossier for a matchup. This prompt is for character research only, not for the final fight verdict.

## 1. Purpose
Build one `character_research.md` file for both fighters.
The output must lock exact versions, collect the best scan-backed evidence, estimate `1-100` stats, and stop before fight analysis.

## 2. Must do
- Lock exact character versions and loadouts first.
- Use source priority in this order:
  1. `reddit.com/r/respectthreads`
  2. `comicvine.gamespot.com/forums/gen-discussion-1/`
  3. `marvel.fandom.com`
  4. `dc.fandom.com`
- Treat Reddit respect threads and ComicVine feat/respect threads as primary proof sources.
- Use Marvel/DC Fandom only for issue lookup, continuity checks, version checks, and page hunting.
- Build candidate pools before choosing final feats or wins.
- Use curated top lists, not exhaustive dumps.
- Keep main lists scan-backed.
- Estimate matchup-relative stats in `1-100`.
- Use backup appendix only for strong issue-backed claims with no usable scan.

## 3. Must not do
- Do not write a base verdict.
- Do not write a conditional verdict.
- Do not simulate the fight.
- Do not add source verdict distribution.
- Do not add a “who wins” section.
- Do not add forum poll synthesis except as support for stat/context reading.
- Do not add extra top-level sections outside the required structure.
- Do not include meta commentary.

## 4. Research selection rules
- Do not stop at the first valid feat or win.
- Read at least 2 proof-oriented sources per character if available.
- Build a candidate pool first.
- Remove wrong-version claims, low-proof summaries, duplicates, and weaker variants.
- Rank candidates by:
  - version purity
  - proof quality
  - magnitude or prestige
  - directness or cleanliness
  - matchup relevance
  - coverage value
- Prefer coverage across at least 3 of these if possible:
  - scale
  - speed
  - durability
  - hax
  - battlefield influence

## 5. List rules
### Feats
- Default `5` per character.
- Expand to `6-8` only if every added item is non-redundant.
- Main-list feats must include:
  - name
  - direct scan/image link
  - exact source: title, issue, page if available
  - short explanation

### Defeated / neutralized opponents
- Default `5` per character.
- Expand to `6-8` only if every added item is non-redundant.
- Prioritize named, high-tier, scan-backed, and clean outcomes.
- Non-clean outcomes must be labeled in the name or explanation.
- Main-list wins/results must include:
  - name
  - direct scan/image link
  - exact source: title, issue, page if available
  - short explanation

### Backup appendix
- Maximum `3` entries per character.
- Use only for strong issue-backed claims with no usable scan.
- Never mix backup items into the main showcase lists.
- Backup items must include:
  - claim name
  - exact issue source
  - short explanation
  - note that no usable scan was found

## 6. Stats
- Use exactly 9 matchup-relative categories.
- Use `1-100`, never `1-10`.
- Estimate per source if needed.
- Map evidence to the 9 categories.
- Average into one final table.
- Round category values to whole numbers.
- Provide exact and rounded overall averages.

## 7. Required output structure
Write `character_research.md` using this exact structure:
- `# Matchup`
- `## 1 Locked versions and loadouts`
- `## 2 Research method`
- `## 3 Character A profile`
- `## 4 Character B profile`
- `## 5 Powers, tools, and weaknesses`
- `## 6 Averaged stats table`
- `## 7 Character A top showcase feats`
- `## 8 Character A top defeated/neutralized opponents`
- `## 9 Character B top showcase feats`
- `## 10 Character B top defeated/neutralized opponents`
- `## 11 Uncertainty notes`

## 8. Quality checks
- Versions/loadouts are explicit and locked.
- Main lists are curated, not exhaustive.
- Main lists use direct scan/image proof.
- Fandom-only claims without usable scans stay out of main lists.
- Backup appendix is small and clearly separated.
- Stats are `1-100` and matchup-relative.
- No fight verdict is included.
